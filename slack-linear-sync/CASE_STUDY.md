# [AI 활용 사례] Slack 질문을 Linear 이슈로 자동화하기 (feat. 귀찮음 해결)

> "질문이 업무가 되는 순간, 기록은 귀찮아진다. 그래서 AI에게 시켰다."

Claude Code와 함께 주말 동안 뚝딱 만든 **Slack → Linear 자동화 봇** 개발기를 공유합니다.

---

## 1. 배경 (Why)

### 문제점: "질문 좀 해도 될까요?" → 기록 증발
우리 팀은 기존에 **여러 채널에 산발적으로 질문이 올라오다 보니**, 정보가 흩어지고 찾기 어려운 문제가 있었습니다. 이를 해결하기 위해 이번에 `#00-ai개발-질문` 채널을 신설하여 질문을 한곳으로 모았지만, 여전히 귀중한 문답들이 휘발되는 문제는 그대로였습니다.

1.  **귀찮음**: 질문하고 답변받기도 바쁜데, 이걸 정리해서 Linear 이슈로 등록하는 건 아무도 안 함.
2.  **컨텍스트 부재**: 누가 어떤 부분에서 자주 막히는지 데이터가 쌓이지 않음. (나중에 팀원 교육이나 온보딩에 쓸 수 있는 자산인데!)
3.  **관리 이원화**: 질문은 Slack에, 태스크는 Linear에 있으니 왔다갔다해야 함.

### 목표
**"Slack에 질문만 남기면, 알아서 Linear 이슈가 되고, 해결되면 이모지 하나로 닫히게 하자."**

---

## 2. 해결 과정 (How)

Claude Code에게 "서버리스 + AI 분석 방식으로 구현해줘"라고 요청했고, 다음과 같은 아키텍처를 잡았습니다.

### 아키텍처
- **Runtime**: Cloudflare Workers (가볍고 빠름, 콜드 스타트 없음)
- **Framework**: Hono
- **Brain**: Claude Haiku 4.5 (질문 내용을 분석해 깔끔한 이슈 제목/설명 생성)
- **Database**: Cloudflare KV (메시지-이슈 매핑 저장)

### 핵심 로직
1.  **감지**: Slack 채널에 메시지가 올라오면 Webhook이 Worker를 깨움.
2.  **분석**: Claude Haiku가 메시지를 읽고 "이 이슈의 제목은 이게 좋겠고, 설명은 이렇게 정리하세요"라고 JSON으로 뱉음.
3.  **매핑**: 이메일을 대조해 Slack 유저와 Linear 유저를 자동 매칭 (Assignee 지정).
4.  **생성**: Linear GraphQL API로 이슈 생성.
5.  **피드백**: 생성된 Linear 링크를 Slack 스레드에 답글로 줌.

---

## 3. 하이라이트 기능 (What's Special)

가장 공들였고, 만족도가 높은 기능 3가지입니다.

### ① `:해결:` 이모지로 원터치 완료
Linear까지 가서 이슈를 닫는 건 너무 귀찮습니다.
그냥 Slack에서 해당 스레드나 메시지에 `:해결:`(또는 `:solved:`) 이모지만 딱 붙이면?
→ **봇이 감지하고 Linear 이슈 상태를 `Done`으로 바꿔줍니다.** (이게 진짜 편함)

### ② 스레드 댓글 동기화
질문에 대한 추가 논의가 Slack 스레드에서 이어질 때, 이 내용이 Linear에는 없어서 맥락이 끊겼었죠.
→ **이제 스레드에 달린 댓글도 자동으로 Linear 이슈의 Comment로 동기화됩니다.**

### ③ AI가 정리해주는 이슈
"이거 안 돼요 ㅠㅠ (에러 로그 스크린샷)" 처럼 대충 질문해도,
→ AI가 **"제목: [Login] AuthError 발생 이슈, 설명: 로그인 시 500 에러 발생, 로그 분석 필요"** 처럼 찰떡같이 정리해서 등록해줍니다.

## 4. 트러블슈팅

개발 중 겪었던 사소한(?) 문제들과 해결책입니다.

- **Linear 공식 봇과 충돌**: 공식 봇도 이슈를 만들고 내 봇도 만들어서 중복 발생.
    - 해결: 쿨하게 공식 봇 연동을 해제하고 커스텀 봇만 사용.
- **원본 링크 누락**: AI한테 "링크 넣어줘"라고 100번 말해도 가끔 까먹음.
    - 해결: AI 프롬프트에 의존하지 않고, 코드 레벨에서 `description += Link`를 강제 추가하는 로직으로 변경 (확실한 통제).

## 5. 결과 및 인사이트

### Before & After
- **Before**: "아 그거 저번에 슬랙 어디서 봤는데..." (검색 실패)
- **After**: Linear에 검색하면 누가 언제 무슨 질문 했고 어떻게 해결됐는지 다 나옴.

### Insight
비정형 데이터(Slack 잡담/질문)를 정형 데이터(Linear 이슈)로 바꾸는 작업에 **LLM(Claude Haiku)**은 가성비 최고의 도구였습니다. Cloudflare Workers와 조합하니 유지비도 거의 0원에 가깝네요.

이제 개발팀은 질문하는데 주저함이 없고, 리더는 팀의 기술적 병목을 한눈에 파악할 수 있게 되었습니다.

> **"자동화는 게으름을 위한 것이 아니라, 더 중요한 일에 집중하기 위한 것이다."**
